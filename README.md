# Analizador de Ventas Comerciales - UTN

Trabajo Práctico Individual para la cátedra de Organización Empresarial 2026.

## 📊 Escenario Seleccionado
* **Escenario B:** Análisis de Ventas de una Pequeña Empresa.
* **Descripción del Dataset:** Registro analítico de ventas y movimientos comerciales (ID, fechas y montos diarios).

## 🗂️ Arquitectura del Repositorio
* `/datos`: Archivo fuente centralizado (`dataset.csv`).
* `/scripts`: Módulo algorítmico desarrollado en Python (`analisis_datos.py`).
* `/resultados`: Repositorio de métricas de rendimiento y gráficos descriptivos.

## 🚀 Guía de Reproducción
Para garantizar la reproducibilidad completa del análisis desde la raíz del proyecto local, corra:
```bash
python scripts/analisis_datos.py
```
Conclusiones
La realización de este trabajo práctico permitió la implementación de un flujo de trabajo profesional basado en metodologías ágiles, integrando herramientas de gestión de proyectos y control de versiones.

Resumen de Actividades
Gestión de Proyectos (Jira): Se estructuró el desarrollo mediante el uso de Issues (SCRUM-1, SCRUM-2 y SCRUM-5), lo que permitió una correcta organización de las tareas y seguimiento de objetivos.

Control de Versiones (Git/GitHub): Se aplicó el concepto de trazabilidad técnica mediante la creación de ramas feature/ para cada tarea. Esto garantizó que cada cambio fuera revisado mediante un proceso formal de Pull Request antes de ser integrado a la rama main, cumpliendo con las buenas prácticas de desarrollo.

Garantía de Calidad (QA): Se llevó a cabo la revisión de código y la documentación final, validando que el script de análisis de ventas e indicadores operara correctamente conforme a los requerimientos establecidos.

Esta experiencia ha sido fundamental para consolidar el uso de ramas, el manejo de commits descriptivos y la importancia de la revisión por pares en entornos de desarrollo colaborativo.
